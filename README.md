## Overview

This dataset provides a unified list of beauty, fragrance, skincare, haircare, and personal care brands. Each entry includes a **URL-friendly slug** (`value`) and a **human-readable display name** (`label`), making it ideal for:

- **E-commerce platforms** — brand filter dropdowns, product tagging, faceted search
- **SEO projects** — brand landing pages, canonical URL generation
- **Content management** — consistent brand naming across a multi-brand site
- **Data normalization** — mapping brand names to standardized identifiers

A select few brands also include rich editorial descriptions, suitable for brand story pages or enhanced product listings.

---

## Data Structure

Each brand entry follows this JSON schema:

```json
{
  "value": "chanel",
  "label": "CHANEL",
  "desc": "Chanel is an iconic French luxury brand..."
}
```

| Field   | Type   | Description                                          |
|---------|--------|------------------------------------------------------|
| `value` | string | URL-safe slug identifier (kebab-case)                |
| `label` | string | Human-readable display name (may include HTML entities) |
| `desc`  | string | Optional brand description (HTML formatted, can be empty) |

## Usage

### As a JSON Array

The `readme` file contains the raw data in this format:

```json
"supportBrands": [
  { "value": "chanel", "label": "CHANEL", "desc": "..." },
  { "value": "dior", "label": "Dior", "desc": "..." }
]
```

### In Frontend Code

```js
// Import and use as a brand selector
const brands = supportBrands.map(b => ({
  slug: b.value,
  name: b.label,
  description: b.desc || null
}));

// Generate brand URLs
const brandUrl = (slug) => `/brands/${slug}`;
```
